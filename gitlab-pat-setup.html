GitLab PAT
등록 가이드

사내 GitLab(gitlab.nexon.com)에서 코드를 내려받을 수 있도록 Personal Access Token을 발급하고 로컬 컴퓨터에 등록하는 과정을 비개발자 기준으로 안내합니다. Windows 사용자는 0단계 WSL 설치를 먼저 끝낸 뒤 이 페이지로 오세요.

Mac Windows (WSL) 비개발자 친화
💡
왜 필요한가요?
사내 GitLab은 보안 정책상 GitLab 비밀번호로는 코드 다운로드(clone)가 되지 않습니다. 대신 Personal Access Token이라는 비밀번호 대용 키를 발급받아 사용해야 합니다. 한 번만 등록해두면 이후에는 자동으로 인증됩니다.

여기서 만든 토큰 1개로 로컬 git용과 나중에 GitLab MCP 연결용까지 모두 재사용할 수 있습니다. 안전한 곳에 보관하세요.
Part 1

GitLab에서 Personal Access Token 발급

Mac·Windows 공통 단계입니다. 웹 브라우저만 있으면 됩니다.

Step 1
GitLab 로그인

브라우저에서 https://gitlab.nexon.com에 접속해서 사내 계정으로 로그인합니다.

Step 2
Access Tokens 메뉴 열기
  1. 화면 오른쪽 위의 본인 프로필 아바타(동그란 아이콘)를 클릭합니다.
  2. 펼쳐지는 메뉴에서 Edit profile (또는 Preferences)를 클릭합니다.
  3. 새로 열린 페이지의 왼쪽 사이드바에서 Access tokens를 클릭합니다.
ℹ️
사이드바가 안 보이면 화면이 좁아서 접혀 있는 겁니다. 왼쪽 위 햄버거(≡) 아이콘을 누르면 펼쳐집니다.
Step 3
새 토큰 만들기
  1. Add new token (또는 + 버튼)을 클릭합니다.
  2. 아래 항목들을 채웁니다.
항목 입력값
Token name 알아보기 쉬운 이름. 예: local-claude-code
Expiration date 달력에서 가장 먼 미래 날짜(보통 1년 뒤)를 선택합니다. 너무 짧게 잡으면 금방 만료돼서 다시 발급해야 합니다.
Select scopes 아래 체크박스를 모두 체크합니다 →
api, read_api, read_user, read_repository, write_repository

마지막으로 Create personal access token 버튼을 클릭합니다.

Step 4
토큰 복사 — 단 한 번만 보입니다

생성된 토큰이 화면 위쪽에 xxxxxxxxxxxxxxxxxxxx 형태로 표시됩니다. 옆의 복사(Copy) 버튼을 눌러 클립보드에 복사하세요.

⚠️
이 화면을 닫으면 토큰을 다시는 볼 수 없습니다. 닫기 전에:
메모장·1Password·키체인·메모 앱에 반드시 붙여넣어 안전하게 보관하세요.
잊어버렸다면 새 토큰을 다시 만들면 됩니다 (예전 토큰은 Revoke로 삭제).
여기부터 OS별로 갈라집니다 · 내 컴퓨터
Part 2-A Mac

Mac에서 토큰 등록하기

macOS에 내장된 Keychain(키체인)에 토큰을 자동 저장하게 설정합니다. 터미널(Terminal) 앱을 열고 진행하세요.

Step 1
터미널 열기

Cmd + Space (Spotlight) → terminal 입력 → 엔터.

Step 2
git 설치 확인
bash 
git --version

버전이 출력되면 OK. "command not found" 또는 설치 안내 창이 뜨면 Install을 눌러 Xcode Command Line Tools를 설치합니다 (5~10분).

Step 3
Credential Helper 설정 — Keychain 사용
bash 
git config --global credential.helper osxkeychain

에러 메시지가 없으면 성공입니다 (아무것도 출력되지 않는 게 정상).

Step 4
이름·이메일 등록 (한 번만)
bash 
git config --global user.name "홍길동"
git config --global user.email "hong.gildong@nexon.com"

사내 이메일과 본인 이름으로 바꿔 입력하세요.

Step 5
테스트 clone — 토큰 첫 등록
bash 
mkdir -p ~/workspace && cd ~/workspace
git clone https://gitlab.nexon.com/projecta/bunit/ai/bbot-skills.git

두 가지 질문이 나옵니다:

prompt 
Username for 'https://gitlab.nexon.com': <여기에 GitLab 사용자명 (이메일 아님)>
Password for 'https://...@gitlab.nexon.com': <여기에 PAT 토큰 붙여넣기>
🎉
다운로드가 시작되면 성공! 다음번부터는 Username·Password를 묻지 않습니다. macOS Keychain이 토큰을 기억합니다.
Part 2-B Windows · WSL

Windows(WSL Ubuntu)에서 토큰 등록하기

0단계 WSL 설치 가이드를 끝낸 분이 대상입니다. 모든 명령은 시작 메뉴 → Ubuntu를 열어서 그 검은 창 안에서 실행합니다. PowerShell이 아닙니다.

Step 1
git 설치 확인
bash 
git --version

버전이 안 나오면 아래 명령으로 설치합니다.

bash 
sudo apt update && sudo apt install -y git
Step 2
Credential Helper 설정 — 파일 저장
bash 
git config --global credential.helper store
📝
store 방식은 토큰을 ~/.git-credentials 파일에 저장합니다. 회사 PC에서 본인만 사용하는 환경이면 충분히 안전합니다.
Step 3
이름·이메일 등록 (한 번만)
bash 
git config --global user.name "홍길동"
git config --global user.email "hong.gildong@nexon.com"
Step 4
테스트 clone — 토큰 첫 등록
bash 
mkdir -p ~/workspace && cd ~/workspace
git clone https://gitlab.nexon.com/projecta/bunit/ai/bbot-skills.git

두 가지 질문이 나옵니다:

prompt 
Username for 'https://gitlab.nexon.com': 
Password for 'https://...@gitlab.nexon.com':
💡
WSL·Ubuntu 창에서 붙여넣기는 마우스 오른쪽 클릭 또는 Shift+Insert입니다. Ctrl+V는 안 될 수 있습니다.
비밀번호 입력 시 화면에 아무것도 안 보이는 건 정상입니다. 안 보여도 입력은 되고 있어요.
🎉
다운로드가 진행되면 성공! 다음번부터는 자동으로 인증됩니다.
FAQ

자주 묻는 질문

"Authentication failed" 에러가 납니다.
Username 자리에 이메일을 넣었거나, Password 자리에 GitLab 비밀번호를 넣은 경우가 대부분입니다.
Username = GitLab 사용자명(프로필 username), Password = 액세스 토큰입니다. 다시 시도해 보세요.
그래도 문제가 발생하면 레포지토리 권한 문제일 수 있으니 #ai-활용-tf 채널에서 문의해 주세요.
토큰을 잃어버렸어요.
Part 1을 처음부터 다시 진행해서 새 토큰을 발급하세요. 예전 토큰은 같은 화면에서 Revoke 버튼으로 삭제하면 됩니다.
토큰이 만료됐다는 에러가 납니다.
새 토큰을 발급한 뒤:
Mac은 키체인 접근에서 gitlab.nexon.com 항목을 삭제.
WSL/Linux는 rm ~/.git-credentials로 파일을 지웁니다.
그 다음 다시 clone하면 새 토큰을 묻습니다.
회사 망 밖(집·카페)에서도 되나요?
사내 VPN에 연결돼 있어야 gitlab.nexon.com에 접근할 수 있습니다. VPN 연결을 먼저 확인해 주세요.
