Step 0 · Windows 전용

WSL 설치 가이드

Windows 컴퓨터를 처음 쓰는 비개발자를 기준으로 작성되었습니다. 한 줄씩 그대로 따라하시면 됩니다.

Mac 사용자는 이 단계를 건너뛰고 바로 1단계. GitLab PAT 등록 가이드로 이동하세요.

Windows 10/11 WSL 2 Ubuntu 비개발자 친화

WSL이 뭔가요?

WSL(Windows Subsystem for Linux)은 윈도우 안에서 리눅스를 쓸 수 있게 해주는 윈도우 기본 기능입니다. 앞으로 설치할 Claude Code, git, 그리고 사내 도구들은 대부분 "리눅스 환경"을 전제로 만들어져 있어요. 걱정 마세요. 새 컴퓨터를 사거나 윈도우를 지우는 게 아닙니다. 윈도우 안에 작은 우분투(Ubuntu) 리눅스를 하나 깔아두는 거예요. 윈도우는 그대로 쓰면서, 필요할 때만 리눅스 창을 띄우면 됩니다.

설치 전 확인 사항

  • Windows 10 (버전 2004 이상) 또는 Windows 11이어야 합니다. — 확인: 시작 메뉴 → 설정 → 시스템 → 정보 → "Windows 사양" → "버전"
  • 인터넷 연결이 되어 있어야 합니다.
  • 회사 보안 정책상 가상화(Virtualization)가 막혀 있지 않아야 합니다. 보통은 켜져 있습니다.
Steps 1 – 3

WSL 설치

PowerShell에서 명령 한 줄을 실행하고 재부팅합니다. 5~15분 정도 소요됩니다.

Step 01
PowerShell을 "관리자 권한"으로 열기
  1. 화면 왼쪽 아래 시작 버튼(윈도우 로고)을 클릭합니다.
  2. 그대로 키보드로 powershell이라고 타이핑합니다.
  3. 검색 결과에서 Windows PowerShell을 마우스 오른쪽 버튼으로 클릭합니다.
  4. 메뉴에서 "관리자 권한으로 실행"을 선택합니다.
  5. "이 앱이 디바이스를 변경할 수 있도록 허용하시겠습니까?" 창이 뜨면 예를 클릭합니다.

반드시 "관리자 권한으로 실행"으로 열어야 합니다. 그냥 클릭해서 열면 다음 단계에서 권한 오류가 납니다.

Step 02
WSL 설치 명령 실행

파란색(또는 검은색) 창이 열리면 아래 명령을 그대로 복사해서 붙여넣고 Enter를 누릅니다.

powershell 
wsl --install

자동으로 아래 세 가지가 한꺼번에 설치됩니다.

  • WSL 본체
  • 가상 머신 플랫폼
  • 기본 리눅스(Ubuntu) 배포판

보통 5~15분 소요됩니다. 진행률 표시가 멈춘 것처럼 보여도 끝까지 기다려 주세요.

Step 03
컴퓨터 재부팅

설치가 끝나면 "재부팅이 필요합니다" 같은 메시지가 표시될 수 있습니다. 작업하던 파일을 모두 저장한 뒤 컴퓨터를 재부팅합니다.

Steps 4 – 6

Ubuntu 초기 설정

계정을 만들고 기본 도구를 설치합니다.

Step 04
Ubuntu 첫 실행 — 사용자 이름과 비밀번호 만들기

재부팅 후 자동으로 검은 창이 열리면서 설치가 완료됩니다. 창이 자동으로 열리지 않으면 시작 메뉴에서 Ubuntu를 검색해서 직접 실행하세요.

잠시 후 아래 질문이 나타납니다.

ubuntu 
Enter new UNIX username:

새 리눅스 사용자 이름을 입력합니다. 윈도우 계정과 달라도 됩니다.

  • 영어 소문자만 사용 (예: hyungrok, taerim)
  • 숫자와 언더바(_)는 OK, 띄어쓰기와 한글은 NO

이후 비밀번호를 두 번 입력하라고 합니다.

비밀번호를 입력할 때 화면에 글자나 별표(*)가 보이지 않습니다. 이건 고장이 아니라 리눅스의 기본 동작입니다. 보지 말고 차분히 입력하세요.

비밀번호는 잘 적어두세요. 앞으로 sudo 명령을 사용할 때마다 가끔 물어봅니다.

Step 05
Ubuntu 최신 상태로 업데이트

Ubuntu 창이 열린 상태에서 아래 명령을 한 줄씩 입력하고 Enter를 누릅니다.

bash 
sudo apt update

비밀번호를 물어보면 4단계에서 만든 비밀번호를 입력합니다 (역시 화면에 안 보입니다). 끝나면:

bash 
sudo apt upgrade -y

2~10분 정도 걸릴 수 있습니다. 끝까지 기다려 주세요.

Step 06
자주 쓰는 도구 미리 설치

앞으로 필요한 기본 도구들을 설치합니다.

bash 
sudo apt install -y git curl wget build-essential
  • git — 사내 GitLab에서 코드를 받아오는 도구
  • curl, wget — 인터넷에서 파일을 받는 도구
  • build-essential — 일부 패키지 설치 시 필요한 컴파일 도구 모음
Step 7

VS Code WSL 연동

코드와 폴더를 다룰 때 사용하는 편집기입니다. WSL 안의 폴더를 직접 열 수 있도록 확장을 설치합니다.

Step 07
VS Code 설치 및 WSL 확장 추가
  1. VS Code 공식 사이트(code.visualstudio.com)에서 Windows 버전을 다운로드하여 설치합니다.
  2. VS Code를 열고, 왼쪽 사이드바에서 네모 4개 모양 아이콘(Extensions)을 클릭합니다.
  3. 검색창에 WSL을 입력합니다.
  4. Microsoft가 만든 "WSL" 확장을 찾아서 Install을 클릭합니다.

설치가 끝나면 VS Code 왼쪽 아래에 >< 모양의 초록 버튼이 생깁니다. 그걸 누르고 Connect to WSL을 선택하면 VS Code가 WSL 환경에 접속합니다.

창 왼쪽 아래에 WSL: Ubuntu라고 표시되면 연결 성공입니다.

설치 확인

Ubuntu 창이나 PowerShell 창에서 아래 명령으로 확인합니다.

powershell 
wsl --status

또는 Ubuntu 창 안에서:

bash 
uname -a
git --version

버전 정보가 정상적으로 출력되면 WSL 설치가 완료된 것입니다.

Step 8

넥슨 인증서 설치

WSL은 새 독립 머신을 실행하는 것과 유사하므로, 사내 인증서를 별도로 등록해야 합니다.

Step 08
인증서 다운로드 및 WSL에 적용

먼저 사내 공유 페이지에서 nexon-cert.zip을 다운로드하고 압축을 해제합니다. Downloads 폴더 안에 nexon-cert/NEXON.pem과 nexon-cert/NEXON.crt 파일이 생성됩니다.

{사용자명} 부분을 반드시 본인의 Windows 사용자명으로 바꿔야 합니다. 사용자명 확인 방법: Ubuntu 창에서 ls /mnt/c/Users 명령을 실행하면 목록이 나옵니다.

VS Code에서 WSL 터미널을 열고 아래 명령을 순서대로 실행합니다.

bash 
sudo apt update
sudo apt upgrade -y

# {사용자명}을 본인 Windows 사용자명으로 변경하세요
# "ls /mnt/c/Users" 명령으로 확인할 수 있습니다
sudo cp /mnt/c/Users/{사용자명}/Downloads/nexon-cert/NEXON.crt /usr/local/share/ca-certificates/
sudo cp /mnt/c/Users/{사용자명}/Downloads/nexon-cert/NEXON.pem /usr/local/share/ca-certificates/
sudo update-ca-certificates
sudo dpkg-reconfigure ca-certificates

echo "export NODE_EXTRA_CA_CERTS=/etc/ssl/certs/ca-certificates.crt" >> ~/.vscode-server/server-env-setup
echo "export NODE_EXTRA_CA_CERTS=/etc/ssl/certs/ca-certificates.crt" >> ~/.zshrc
echo "export NODE_EXTRA_CA_CERTS=/etc/ssl/certs/ca-certificates.crt" >> ~/.bashrc
echo "export REQUESTS_CA_BUNDLE=/etc/ssl/certs/ca-certificates.crt" >> ~/.vscode-server/server-env-setup
echo "export REQUESTS_CA_BUNDLE=/etc/ssl/certs/ca-certificates.crt" >> ~/.zshrc
echo "export REQUESTS_CA_BUNDLE=/etc/ssl/certs/ca-certificates.crt" >> ~/.bashrc
echo "export SSL_CERT_FILE=/etc/ssl/certs/ca-certificates.crt" >> ~/.vscode-server/server-env-setup
echo "export SSL_CERT_FILE=/etc/ssl/certs/ca-certificates.crt" >> ~/.zshrc
echo "export SSL_CERT_FILE=/etc/ssl/certs/ca-certificates.crt" >> ~/.bashrc
echo "export NODE_TLS_REJECT_UNAUTHORIZED=0" >> ~/.vscode-server/server-env-setup
echo "export NODE_TLS_REJECT_UNAUTHORIZED=0" >> ~/.zshrc
echo "export NODE_TLS_REJECT_UNAUTHORIZED=0" >> ~/.bashrc

# 여기까지 입력 후 VS Code 종료
sudo reboot

재부팅 후 PowerShell에서 아래 명령으로 WSL을 재시작합니다.

powershell 
# 설치된 WSL 환경 이름 확인
wsl -l -v

# Docker 관련 제외, 설치된 이름이 Ubuntu라면
# 아래 명령으로 WSL 재실행
wsl -t Ubuntu
wsl -d Ubuntu

# 1분 후 VS Code 실행
FAQ

자주 묻는 질문

윈도우 폴더(C:\Users\...)와 WSL 폴더는 어떻게 구분되나요?

WSL Ubuntu 창에서 cd ~를 입력하면 "리눅스 홈"으로 이동합니다 (/home/사용자이름). 윈도우의 C 드라이브는 /mnt/c/로 접근할 수 있습니다.

앞으로 작업할 폴더는 가급적 리눅스 홈 안에 만드세요 — 훨씬 빠르고 안정적입니다.

Ubuntu 창을 나중에 다시 어떻게 열 수 있나요?

시작 메뉴에서 Ubuntu를 검색해서 클릭하면 됩니다. 또는 PowerShell에서 wsl만 입력해도 됩니다.

비밀번호를 잊어버렸습니다.

PowerShell(관리자)에서 wsl -u root로 root 권한으로 접속한 뒤, passwd 사용자이름으로 재설정할 수 있습니다.

wsl --install이 "가상화가 비활성화되어 있습니다" 에러를 냅니다.

PC의 BIOS에서 가상화(Virtualization, VT-x, AMD-V)가 꺼져 있는 경우입니다. 회사 IT 담당자에게 "BIOS에서 가상화 켜주세요"라고 요청해 주세요.

VS Code에서 터미널 실행 시 chdir(2) failed.: Permission denied 문제가 발생합니다.

PowerShell에서 WSL을 재시작하면 해결됩니다.

powershell 
# 설치된 WSL 환경 이름 확인
wsl -l -v

# 설치된 이름이 Ubuntu라면 아래 명령으로 재시작
wsl -t Ubuntu
wsl -d Ubuntu

# 이후 PowerShell 재실행 후 WSL로 접근
wsl

# 이후 VS Code를 실행하여 터미널 접근
